# Plans

> **MANDATORY:** Before implementing any non-trivial feature, a plan document must exist in this folder.
> A plan covers **what** and **why** — requirements, scope, constraints, and acceptance criteria.
> The **how** and **in what order** belongs in `refdocs/execution/`.

## Naming convention
`PLAN_<FEATURE_NAME>.md` — e.g. `PLAN_WEB_V2_PORT.md`

## Required sections in every plan
1. **Goal** — one paragraph on what this achieves and why it matters
2. **Scope** — what's in and explicitly what's out
3. **Constraints** — tech limits, time, dependencies
4. **Acceptance criteria** — how you know it's done
5. **Open questions** — things still to decide before execution starts
